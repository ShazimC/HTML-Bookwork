Chapter 7: Forms

Form Structure

Form tag: Form controls live inside the form element. This element should always carry the action attribute and will usually have a method and id attribute too.
Action attribute: Every form element requires an action attribute. Its value is the URL for the page on the server that will receive the information in the form when it is submitted.
Method attribute: Forms can be sent using on of two method values: get or post.
Get value: With the get method, the values from the form are added to the end of the URL specified in the action attribute. The get method is ideal for:

  • short forms (such as search boxes)
  • when you are just retrieving data from the web server (not sending information that should be added to or deleted from a database).
Post value: With the post method, the values are sent in what are known as HTTP headers. As a rule of thumb you should use the post method if your form:
  • allows users to upload a file.
  • is very long.
  • contains sensitive data (passwords, pins etc).
  • adds information to, or deletes information from, a database.
Note: If the method attribute is not used, the form data will be sent using the get method by default. ID attribute: The id attribute is looked in detail at page 183 but the value is used to identify the form distinctly from other elements on the page (and is often used by scripts - such as those that check if you have entered information into fields that require values).

Text Input:

Input tag: The input element is used to create several different form controls. The value of the type attribute determines what kind of input they will be creating.
name: When users enter information into a form, the server needs to know which form control each piece of data was entered into. Therefore, each form control requires a name attribute. It is for better communication with the server.
size: The size attribute should not be used on new forms. It was used in older forms to indicate the width of the text input (measured by the number of characters that would be seen.) For example a size of 3 would create a box wide enough to display at max of three characters.
maxlength: You can use maxlength attribute to limit the number of characters a user may enter into the text field. Its value is the max number of characters that can be entered into the box.

Drop Down List Box

  • select element: a drop down list box also known as a select box allows users to select one option from a drop down list. The select element is used to create a drop down list box. It contains two or more option elements.
  • option element: is used to specify the options that the user can select from. The words between the opening option and closing option tags will be shown to the user in the drop down box.
  • selected attribute: can be used to indicate the option that is selected by default.

Button and Hidden Controls

The hidden type allows the web author to hide the form controls from the page and therefore the user. This allows them to add values to forms that the user cannot see. For example the web page author might use a hidden field to indicate which page the user was on when they submitted a form.

Labelling Form Controls

label element: Each from control should have its own label element as this makes the form accessible to vision-impaired users.
The label element can be used in two ways. It can:

  1. Wrap around both the text description and the form input.
  2. Be kept seperate from the form control and use the for attribute to indicate which form control it is a label for.
for attribute: The for attribute states which form control the label belongs to. Note how the radio buttons use the id attribute. The value of the id attribute uniquely identifies an element from all other elements on the page. You match the for value with the id value.


Tips for placing labels:

    To the above or left of:
  • Text Inputs
  • Text Areas
  • Select boxes
  • File Uploads
    To the right of:
  • Individual checkboxes
  • Individual radio boxes.

Grouping form elements

fieldset: You can group related form controls together inside the fieldset element. This is particularly helpful for longer forms. The Appearance of the lines can be adjusted using CSS.
legend: The legend element can come directly after the opening fieldset tag and contains a caption which helps indentify the purpose of that group of form controls.

HTML5: Form Validation

Validation helps ensure the user enters information in a form that the server will be able to understand and process when the form is submitted. It helps reduce the amount of work the server has to do, and enables the user to see if there are problems witht he form faster than if validation were performed on the server.
Use the attribute required with value required to inform the user to fill out the field.
